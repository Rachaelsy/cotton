# AI QA MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a usable AI Q&A module for cotton farmers with real text chat, intent jump cards, local chat history, bilingual prompts, and honest voice/TTS capability boundaries.

**Architecture:** Keep the existing `server/routes/ai.js` proxy as the backend boundary, but add a shared intent parser in `utils/ai-intent.js` so backend tests and mini-program behavior agree. The front end continues to use `pages/ai/*`, rendering optional jump cards returned by `/api/ai/chat` and preserving recent chat locally.

**Tech Stack:** WeChat Mini Program JavaScript/WXML/WXSS, Express routes, DeepSeek/Groq/Siliconflow OpenAI-compatible chat APIs, Node `assert` route tests.
